# Webについて
---
### プロトコルとは何か説明してください。

- コンピュータ同士の通信の取り決め
- 通信を行うための共通の規格として用いる
- 別々の機器同士でも通信を行える
- IP（インターネット・プロトコル）：特定の機器同士を接続するためのプロトコル
- TCP（トランスミッション・コントロール・プロトコル）：安全にデータを通信するためのプロトコル
- SMTP（シンプル・メール・トランスファー・プロトコル）：メール形式でデータをやり取りするためのプロトコル
- HTTP（ハイパー・テキスト・トランスファー・プロトコル）：Web サイトを表示するためのプロトコル

### httpとhttpsの違いについて説明してください。

- https（ハイパー・テキスト・トランスファー・プロトコル・セキュア）：通信がSSLによって、暗号化されているHTTP
- SSL/TPS による暗号化通信は、一対の機器の間でデータの暗号化通信を行うことができるトランスポート層のプロトコル
- https は、対になる2つの鍵「共通鍵暗号方式」「公開鍵暗号方式」の仕組みで行われる

### Webサーバがどのような役割を持っているかについて説明してください。

- Web サーバは、HTTPリクエストを受け取り、リクエストに応じたHTTPレスポンスを返す
- URLを入力すると、ブラウザから Web サーバに HTTP リクエストが送信される

### クライアントとサーバの違いについて説明してください。

- サーバ：サービスを提供するコンピュータ
- クライアント：サービスの提供を受けるコンピュータ
- サーバとしてのスペックとして、大量のリクエスト、長時間稼働を要する


### URLにアクセスして動的なウェブサイトの画面が表示されるまでの流れを説明してください。

- クライアントからWebサーバに、HTTP リクエストが送られる
- Web サーバはアプリケーションサーバに対して、リクエスト処理を行う
- アプリケーションサーバはデータベースサーバに対して、SQL文を実行する
- データベースサーバはアプリケーションサーバに、データを返す
- アプリケーションサーバは web サーバに、実行結果を返す
- web サーバはブラウザに、HTTPレスポンスを返す
- 返ってきた HTTP レスポンスを、ブラウザが解析して画像表示する

# Apacheについて
---
### Apacheとはどのようなソフトウェアか説明してください。

- 無償
- Web サーバソフトウェア
- Web サーバとして機能させたいコンピュータ内にインストールして使用する
- 複数のコンテナ同士の組み合わせで、擬似的な仮想環境を構築することができるソフトウェア

### DocumentRootがどのような役割を持ったディレクティブか説明してください。

- Web 上に公開するサーバ内のディレクトリを設定するディレクトリ
- 初期では /var/www/html という値が設定されている
- どのディレクトリにファイルを探しに行くのか、という指定をしている

### DirectoryIndexがどのような役割を持ったディレクティブか説明してください。

- ファイルの名前を設定する役目
- ex) DirectoryIndex index.html
- DocumentRoot ディレクティブは、ディレクトリの指定
- DirectoryIndex ディレクティブは、ファイルの指定をしている


# Dockerについて
---
### Dockerとはどのようなソフトウェアか説明してください。

- コンテナ型の仮想環境を作成、配布、実行するためのソフトウェア
- アプリケーションや実行環境をプロセスとして、コンテナという単位で起動
- 仮想マシンと比べて、仮想OSを作る必要がないため、時間と労力がかからない
- 起動・破棄などの操作が容易・高速である
- 構築する環境はすべてコードベースで管理される
- 環境を開発チームに共有したり、環境差分を解消できる 
- Docker は単一のコンテナを取り扱う
- 複数のコンテナに対して、起動・停止・破棄を行うには不便

### Docker-Composeとはどのようなソフトウェアか説明してください。

- Docker-Compose は複数のコンテナを、一括で操作ができる
- docker-compose のコマンドは、docker-compose.yml を参照している

### コンテナとはどのようなものか説明してください。

- 使い捨てされる
- コンテナ内で行った、作業、データ処理などは、コンテナ破棄時にすべて破棄される
- 仮想OSを必要としない、仮想環境
- コンテナの設計図を、image が担っている

### Dockerにおけるイメージとはどのようなものか説明してください。

- コンテナの設計図にあたるものがイメージ
- 世の中にはたくさんの種類のイメージがある
- DockerHub に公開されている
- 基本的には DockerHub に公開されているベースイメージを使う
- 公式イメージをカスタマイズして、自作イメージを DockerHub に公開できる

### DockerにおいてDockerfileは何のために使用するか説明してください。

- ベースとなるイメージを拡張し、新たに作成する自作イメージの内容を定義する設計書として使用
- 起動するコンテナのベースイメージを指定する
- そのイメージをベースとしてコンテナを起動するタイミングで必要なパッケージをインストールする
- 設定ファイルを配置するといったコマンドを記述 

### Dockerのボリュームマウントは何のために使用するか説明してください。

- コンテナ内のデータを保存するのが volume
- volume を使用すると、Docker ホストが指定したディレクトリとコンテナの指定したディレクトリを共有できる
- コンテナは破棄されるが、Docker ホスト側にデータが残る
- 初めからホストOSに用意してあるファイルやディレクトリをコンテナ内で共有するためにも使用


# ネットワークについて
---
### ドメインとは何か説明してください。

- IP アドレスのような数字の羅列は人間にとって覚えにくい。。。
- 直感的にどこの Web サイトのアドレスかわからない。。。
- ドメイン直感的にどこの Web サイトかわかる！人間にとって覚えやすい
- ドメインから IP アドレスを導き出す仕組みがある！→【 DNS 】

### ホスト名とは何か説明してください。

- 文書を格納するコンピュータのドメイン名や IPアドレス を指定していること
- ホスト名 = ホスト（www） + ドメイン(〇〇.com)
- ホスト：ホストコンピュータを意味する

### ポートとは何か説明してください。

- IP アドレスが住所なら、ポートは玄関
- コンピュータとインターネットの間にあるドア
- IP アドレス + ポート番号で通信を行う
- インターネットサーバ側のポート番号には大きな意味を持つ
- データの種類によって、通るドアが変わる
- インターネットサーバ側のポートは、固定の番号となっており、常に扉が開いた状態になっている
- http://~ はすべて80番ポートが入口（デフォルト）
- ポート番号によって、サーバの機能（サービス）がちがう

### ポートフォワードとは何か説明してください。

- 特定のポートにやってきたデータを中継すること
- ex) A君のパソコンの1000番のポートにやってきたデータを、B君のパソコンの2000番ポートに転送する
- 特定のポートにやってきたデータを、別のところに送ること

### 127.0.0.1というIPアドレスが何か説明してください。

- 自分でーす、と意味するIPアドレス
- ループバックアドレス
- そもそもIPアドレスは、コンピュータのネットワーク上の住所
- よく使われるループバックアドレスのひとつ
- ～127.255.255.254 ループバックアドレス
- 127.0.0.0 はネットワークアドレス
- 127.255.255.255 はブロードキャストアドレス
- IP アドレスは、192.168.1.2 みたいな感じ

### localhostとは何か説明してください。

- 自分でーす、を意味するホスト名
- ネットワークでつながっている自分方のコンピュータのこと
- 相手方のコンピュータを「リモートホスト」
- http://localhost/：自分のコンピュータに接続
